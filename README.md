Необходимо создать «телефонную книгу» для локальной сети, чтобы к устройствам можно было обращаться по понятным именам,
например printer.office.lan.
Кроме того, вы должны обеспечить обработку локальных запросов нашим DNS-сервером, а запросы к внешним ресурсам (например, google.com)
должны перенаправляться на публичный DNS-сервер.
При запросе к @127.0.0.1 локальное имя printer.office.lan должно возвращать 192.168.1.100, в то время как внешнее имя,
например google.com, должно возвращать один или несколько публичных IP-адресов.
